Pyodide Loading Flow

  1. User opens the page. A loading spinner shows "Loading scientific engine..."
  2. A web worker starts downloading Pyodide + sympy + astropy + numpy (~30MB). The UI stays responsive — the sidebar, search, and autocomplete all work from the bundled JSON, so the user can browse constants even before Pyodide is ready.
  3. Once loaded (~3-8s cold, instant on subsequent visits thanks to service worker cache), the spinner disappears. The expression editor becomes active.
  4. When the user hits Cmd+Enter, the expression is sent to the worker via postMessage. The worker runs the existing Python AstroCalculator.calculate() and returns the result.
  5. The result is displayed in the result cards. The editor remains editable during computation.

Deployment (GitHub Pages)

Vite builds to dist/ — static HTML + JS + WASM assets. A GitHub Action runs on push to main:

  1. Checkout + install deps
  2. Run dump-constants.py and dump-units.py to regenerate JSON
  3. Run vite build
  4. Deploy to gh-pages branch (via peaceiris/actions-gh-pages or similar)

Users access it at https://<username>.github.io/astrocalculator-web/.

What's Preserved from the CLI

  • Same expression syntax — implicit multiplication, ^ for exponentiation, in <unit>
  • Same AstroCalculator Python class — identical calculation results
  • Variable assignments with comma/multi-line separation
  • SI + CGS result display
  • Access to all 28+ astropy constants
  • History (now via localStorage instead of disk)

What's New

  • Searchable constants/units tables always visible in sidebar
  • Autocomplete of constants and units in the editor
  • Equation templates with one-click insertion
  • KaTeX-rendered equations
  • Multi-line text editing with proper cursor control
  • Keyboard shortcuts (Cmd+Enter, Cmd+K, Cmd+J)
  • Copy buttons on results
  • Accessible from any device (mobile/tablet via browser)

Design — Section 4: Equations Folder & Data Flow

Equation templates are defined as markdown files in a project folder, loaded at build time, and rendered as clickable templates in the sidebar.

Equations Folder Structure

equations/
├── escape-velocity.md
├── schwarzschild-radius.md
├── kepler-third-law.md
├── eddington-luminosity.md
├── free-fall-time.md
└── ...

Markdown File Format

Each file uses YAML frontmatter for structured data + markdown body for the rendered LaTeX description:

--- title: Escape Velocity category: Mechanics tags: [velocity, gravity, compact objects] params: - symbol: M default: "1.4 M_sun" description: Mass of the object - symbol: R default: "10 km" description: Radius expression: "sqrt(2 G M / R) in km/s" --- The escape velocity from a body of mass $M$ and radius $R$ is: $$v_{\rm esc} = \sqrt{\frac{2GM}{R}}$$ For a neutron star ($1.4\,M_\odot$, $10\,{\rm km}$), $v_{\rm esc} \approx 0.6c$.

How It Works

1. Build-time ingestion

A Vite plugin or build script reads all .md files from equations/, parses the YAML frontmatter, and generates a JSON manifest bundled into the app. No runtime file loading needed.

2. Sidebar rendering

The Equations tab shows a searchable list. Each item displays the title + rendered LaTeX equation (via KaTeX). Clicking expands it to show parameter descriptions.

3. "Add" button

Clicking Add takes the params (as variable assignments — one per line) and the expression, and inserts them into the editor. The editor's content is replaced (if empty) or appended (if already has content).

Constants JSON (build-time snapshot)

A build script runs astropy to dump all supported constants into src/data/constants.json:

[
  {"symbol":"G","name":"Gravitational constant","value":6.6743e-11,
   "unit":"m^3/(kg s^2)","uncertainty":1.5e-15,"ref":"CODATA 2018"},
  {"symbol":"c","name":"Speed of light","value":2.99792458e8,
   "unit":"m/s","uncertainty":0,"ref":"CODATA 2018"},
  ...
]

This JSON is what powers the SearchBar, ConstantsTable, and autocomplete — all instant, no Pyodide needed. The Pyodide engine is only invoked when the user evaluates an expression, which gives the authoritative numeric result.

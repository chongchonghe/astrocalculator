Which frontend framework?

GitHub Pages means static output. Pyodide means a heavy WASM dependency. The framework should be lightweight to offset Pyodide's size.

React (with Vite)

Most popular, massive ecosystem. Easy to find components for tables, autocomplete, and equation rendering. Vite keeps the dev experience fast.

  • Huge ecosystem (headless table, combobox, LaTeX renderers)
  • Well-tested with Pyodide in production
  • Easy to hire/contribute
  • ~40KB gzipped (React + ReactDOM)
  • Can be overkill for a single-page calculator

Svelte

Compiles away at build time — no runtime library. Very small bundle. Reactive by default with minimal boilerplate.

  • Tiny output (~2KB overhead)
  • Less JS shipped = more room for Pyodide
  • Simple, readable syntax
  • Smaller ecosystem
  • Fewer Pyodide integration examples

Vanilla JS + Web Components

No framework at all. Use native web components and ES modules. Lightest possible approach.

  • Zero framework overhead
  • No build step complexity
  • No dependency churn
  • More boilerplate for reactive state
  • Must build table/autocomplete from scratch
  • Harder to add features over time

Given your plan to add equation templates, prefilled parameters, and browsable constant tables over time, React is my recommendation — the ecosystem will save you time on each new feature.

Design — Section 3: Main Panel & Expression Editor

The right side of the Studio layout — where the actual calculation happens.

ExpressionEditor

Multi-line expression editor
1 M = 1.4 M_sun
2 R = 10 km
3 sqrt(2 G M / R) in km/s

Multi-line textarea (not single-line input)

A <textarea> that supports multiple lines. Each line is a variable assignment or the final expression. This directly mirrors the current CLI's multi-line input model but with proper text editing (cursor movement, select, copy/paste, undo).

Keyboard shortcuts

  • Cmd+Enter — evaluate expression
  • Cmd+K — focus search bar in sidebar
  • Escape — clear editor

Autocomplete overlay

As the user types, a dropdown appears below the cursor showing matching constants, units, and functions. Selecting one inserts it. This complements the sidebar — you don't need to leave the editor to find a constant. Searches the same JSON dataset as the sidebar.

ResultDisplay

PARSED
sqrt(2*G*M*1/R)
RESULT (SI)
1.9277e+08 m / s
RESULT (CGS)
1.9277e+10 cm / s
IN km/s
1.9277e+05 km / s

Three result cards (Parsed, SI, CGS) displayed after evaluation. If the expression contains an in <unit> clause, a fourth card shows the converted value. Each card has a copy button to copy the value.

HistoryPanel (sidebar tab)

The 4th sidebar tab. Shows the last N calculations (stored in localStorage). Each entry shows the original expression and result. Clicking an entry reloads it into the editor. A "Clear" button at the bottom. No server-side storage — local only, private to the browser.
